# 联机 FPS 改造方案

## 决策（已确认）
- **部署**：前后端合并到单个 Node.js 服务（同源，无 CORS），一条命令启动。GitHub Pages 保留为纯静态镜像。
- **登录**：全功能都需先登录，未登录无法进入游戏（加载即出现登录界面）。
- 单机内容（关卡 PVE、与 AI 的王者乱斗）完整保留，与联机并存。

## 技术选型
| 层 | 选型 | 理由 |
|---|---|---|
| 后端运行时 | Node.js + Express | 同源托管静态文件 + REST + ws |
| 实时通信 | `ws`（原生 WebSocket） | 轻量、可控，避免 socket.io 体积；断线重连自实现 |
| 数据库 | **Better-sqlite3**（文件型 SQLite） | 零配置、随仓库即跑，单文件持久化；适合演示与小规模 |
| 密码加密 | `bcrypt` + crypto 随机盐 | 业界标准 |
| 会话 | 随机 token（crypto.randomBytes）存于服务端 sessions 表 + httpOnly cookie | 无状态前端、可注销 |
| 部署 | `npm start` 本地；Render/Railway 一键部署（给 Dockerfile + 说明） | |

## 后端模块（server/）

**db.js** — 自动建表（首次启动）：`users(id, username, email, passwordHash, createdAt)`、`sessions(token, userId, createdAt)`、`friendships(userA, userB, status, createdAt)`、`friend_requests(from, to, status)`、`messages(id, roomId, fromUser, toUser, content, createdAt)`、`rooms(code, host, mapKey, teamSize, state, createdAt)`、`room_players(roomCode, userId, team, slot)`。

**auth.js** — `POST /api/register`、`POST /api/login`（下发 httpOnly cookie）、`POST /api/logout`、`GET /api/me`。bcrypt 哈希，用户名/邮箱唯一约束，输入校验，错误统一中文提示。

**social.js** — `GET /api/friends/search?q=`、`POST /api/friends/request`、`POST /api/friends/accept`、`POST /api/friends/reject`、`POST /api/friends/remove`、`GET /api/friends`（含在线状态，来自 ws 连接表）。所有操作鉴权。

**realtime.js** — 单一 `ws.Server`。消息类型（JSON）：`hello/auth`、`chat`(私聊/房间/队伍频道)、`presence`、`match`（创建/加入/匹配）、`team`（组队邀请/加入/退出）、`game`(玩家位置快照、动作、弹道、命中、生命值、死亡/复活)、`reconnect`。每个连接绑定 userId，服务端维护 `onlineUsers` Map + 每个房间的玩家集合。聊天与状态都通过 ws 实时推送。

**状态同步模型**（低延迟，权衡复杂度）：
- **权威客户端 + 广播**：每个玩家以 ~20Hz 上报自己的快照（pos/rot/动作/生命值/武器槽/弹药）；服务端仅转发（不去抖、不插值），其他客户端做**快照插值**（buffer 2 帧 + lerp）显示远程玩家。
- **弹道/命中**：射击者本地判定命中 → 发 `hit{targetId, dmg, headshot, crit}` → 服务端转发给目标客户端；目标客户端在自己的权威状态上扣血并回广播。被命中者死亡由其自己客户端判定并广播 `death`。这种"命中者发起、被击者确认"的模型够用且避免复杂服务器碰撞。
- **断线重连**：服务端保留房间 60s；客户端重连时发 `reconnect{roomCode}`，服务端补发房间当前快照（各玩家最新已知位置/生命值）。

## 前端改造（index.html）

复用现有 `window.*` 桥接约定（已有 `runWzlbLoading`/`__wzlbDebug`）。

### 1. 登录/注册界面（新增 `<div id="auth-screen">`）
- 全屏，沿用现有赛博朋克 CSS 变量（`#050810` 背景、霓虹青/红、Orbitron 字体），与启动屏一致。
- 登录/注册切换 Tab，用户名或邮箱 + 密码，错误提示内联。
- 注册成功/登录成功后调用 `onAuthed(user)` 进入游戏。
- **所有原有 JS 在未登录时不执行游戏逻辑**：用一个 `if (!currentUser) return showAuth()` 守卫包住原有的 `registerStartScreenButtons()` 触发与 `init()`。登录前先加载 Three.js（无副作用），登录后才解锁按钮。

### 2. 启动屏新增入口（沿用 `v-dock` 风格）
- 新增 dock 按钮：**好友**（👥）、**聊天**（💬）、**组队**（🛡️ 联机）。
- 王者乱斗弹窗 `#wzlb-modal` 副标题改为支持"联机对战/单人（AI）"二选一：保留原"进入战场(单人)"按钮，新增"**创建联机房间**"和"**快速匹配**"按钮。
- 顶栏显示当前用户名 + 退出登录按钮。

### 3. 好友面板 `#friends-panel`
- 搜索框、好友列表（在线绿点/离线灰）、申请列表（待处理）、删除好友。沿用 `modal-overlay`/`modal-box` 样式。

### 4. 聊天窗口 `#chat-panel`
- 可折叠侧栏，频道切换：私聊（每个好友）/ 房间频道 / 队伍频道。消息实时推送，未读红点。

### 5. 组队面板 `#team-panel`
- 显示当前队伍成员（队长标记）、邀请好友下拉、退出队伍。队长有"带队匹配"按钮。

### 6. 联机对战集成（核心，复用 WZLB）
- 关键洞察：现有 `startMatch()` 创建 `createBot('enemy'/'ally')` 驱动的 AI。**联机模式下用真实远程玩家替代 AI**：
  - 新增 `createRemotePlayer(team, profile)`：复用 `buildBotMesh()`、复用 `createHealthBarSprite()`，但 `isBot:false`、无 AI 字段，位置由网络快照驱动。
  - 远程玩家 mesh 加进 `bots[]`（沿用 minimap/HUD 逻辑）与 `enemies[]`（敌方才进，沿用命中判定）。
  - 在 `animate()` 中新增 `updateRemotePlayers(delta)`：对 `isBot:false` 的 bot，用快照插值更新位置/朝向；命中由网络 `hit` 消息驱动其 `health` 变化，沿用 `onBotKilled`/复活逻辑。
- 客户端 `MultiplayerClient`（新 `<script>` 块，全局 `window.net`）：
  - 连接 ws、鉴权、发送快照、接收并写入 `remoteSnapshots`。
  - 玩家本地射击命中远程玩家时，调用 `net.sendHit(targetId, dmg, headshot, crit)`。
  - 玩家自己移动/换弹/切武器/死亡/复活时定期广播状态。
  - 收到自己被命中：扣血；死亡：`playerDie()`，由现有复活/结算接管。
  - 断线检测（心跳）+ 自动重连（带 roomCode）。
- **单人模式（PVE / AI 王者乱斗）完全不变**：`wzlbActive && matchKind==='ai'` 时走原 `createBot`/`updateBotAI` 路径。

### 7. 界面适配
- 全部新 UI 用现有 CSS 变量与字体，新面板做成半透明玻璃态卡片（`backdrop-filter`）贴合赛博风。
- 响应式：聊天/好友/组队面板默认收起为侧边图标，点击展开；移动端被 `#pc-guard` 拦截（沿用原逻辑）。

## 验收标准
1. 注册→登录→进入主菜单；注销回到登录页；密码 bcrypt 加密；cookie 会话。
2. 搜索/添加/接受/删除好友；好友在线状态实时变化。
3. 私聊、房间频道、队伍频道实时消息；离线消息按房间保留。
4. 联机房间：创建/加入/退出；快速匹配组队；多端真实玩家可见、可互相射击、位置/血量同步、断线 60s 内可重连。
5. 组队：队长带队匹配，队内频道聊天。
6. 单机全部内容仍可玩；未登录被拦截在登录页。
7. `npm start` 一次启动，前后端同源可用。
