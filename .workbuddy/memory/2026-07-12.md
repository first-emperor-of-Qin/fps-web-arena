## 抽奖四大 BUG 修复（v5.6）
- 用户反馈 4 项：(1) 失败/死亡也弹抽奖（应仅胜利弹）；(2) 停留在抽奖页可无限抽（应每关仅一次）；(3) 抽奖显示数量与实际到账不一致；(4) 转速太慢，需先快后慢且总时长 <10s。
- 根因与修复：
  1. `playerDie`(17490) 与王者乱斗结算(28951) 在失败时也调 `_showGachaReward('defeat',...)`。删除 playerDie 抽奖调用；王者乱斗改为 `if(win && ...) _showGachaReward('victory',5)`。战役胜利 `showVictoryScreen`(18921) 抽奖保留。全文件 `_showGachaReward('defeat'` 归零。
  2. 无限抽奖：`spinGacha` 结束时把 `_gachaSpin=false` 但 `core.onclick` 仍绑 → 可反复点。改为入口 `if(_gachaSpin||_gachaDone)return`，转动开始即 `core.onclick=null`，结束置 `_gachaDone=true`（showGacha 每次重置 `_gachaDone=false` 并重绑 onclick）。
  3. 数量不一致：旧逻辑高亮停在 `track[(totalSteps-1)%16]`（winIdx 前一格），发奖却用 `track[winIdx]` → 看到的格与到账不符。重写 tick 为「先高亮后判定」：step===totalSteps 时 idx%16 恰为 winIdx，发奖改用 `track[idx%16].dataset.amt`，保证高亮停留格==显示==到账。结束追加 `_applyCharacterStats()`+`refreshHome()`。
  4. 转速/时长：`laps` 3→2；改为预算制缓动——`budget=6200ms`，每步权重 `w=0.25+t²*3.4`（先快后慢），`delays[i]=max(22, w/sumW*budget)`，实际总和 <budget，严格 <10s。
- 验证（本机 Chrome + puppeteer-core headless swiftshader）：胜利抽奖弹出 OK；转动总时长 6563ms(<10s)；高亮停留格=6、中心显示+6、tip「共6碎片」、localStorage 到账增量=6 → 三者一致；结束后连点 5 次碎片不再增加（无限抽奖已堵）；无 pageerror。CSS 大括号 1180/1180；4 段内联 JS Node --check ALL OK。
