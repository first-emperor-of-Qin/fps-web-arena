# 音频改动说明 · COMBAT STRIKE v5.0

> 本游戏音频 100% 由 Web Audio API **程序化合成**（无任何外部音频文件），因此本次改动直接改写 `index.html` 内的合成逻辑，无需新增资源。

## 一、更换背景音乐（BGM）
原曲为 A 小调、128 BPM、四-on-the-floor 驱动型电子乐。已重写为一条**全新的合成波（synthwave）循环**：

| 维度 | 原曲 | 新曲 |
|---|---|---|
| 调性与进行 | A 小调 Am–F–C–G | **E 小调 i–VI–III–VII（Em–C–G–D）** |
| 速度 | 128 BPM | **132 BPM** |
| 旋律 | 跳跃式方波 lead | 新写方波 lead 旋律线 |
| 鼓点 | 底鼓+军鼓+闭合踩镲 | 底鼓 + 军鼓 + **反拍开镲长音**律动 |
| 新增层 | — | **琶音拨奏层（BGM_ARP）+ 每小节柔和 pad 持续和弦**（氛围底色） |

音频总线层级保持不变：`音符 gain → musicGainNode → masterGainNode → destination`，音乐音量仍受游戏内音乐音量控制。

## 二、为所有武器添加专属射击 / 换弹音效
新增「武器感知」音频引擎：
- `WEAPON_SFX` 配置表：每把武器独立定义 `shoot`（射击）与 `reload`（换弹）音色参数。
- `_sfxNoise / _sfxTone / _sfxShoot / _sfxReload` 通用合成原语。
- 重写 `playShootSound()` 与 `playReloadSound()`：在发声瞬间通过 `getCurrentEquippedWeaponId()` 解析**当前武器**，合成对应专属音色。

> 近战（雷霆战刃/地狱魔镰/千机伞）、投掷（手雷类）、战术（数据魔方）不走射击/换弹逻辑，各自已有独立激活音效，不在本次范围内。

## 三、验证
- 抽取全部 inline `<script>` 运行 `node --check`，**全部语法通过**（含 150 万字符的主游戏脚本）。
- 用 mock Web Audio API 跑运行时测试：11 把武器的 shoot/reload 路径 + 新 BGM 全 64 步调度器 + 6 个专属射击函数回退，**零异常**。

## 四、试听方式
直接用浏览器打开 `index.html` 并进入关卡即可听到新 BGM 与各武器专属音效（需用户交互后浏览器才会启动 AudioContext，符合浏览器自动播放策略）。
